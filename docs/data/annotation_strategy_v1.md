# v1 标注策略

第一阶段只接受确定性弱标注，不再使用随机 mock 标签作为正式训练标签。

## 当前标签定义

任务：`LocalRouteChoice-Lite`

标签来源：

1. 读取真实 KiCad `track segment`。
2. 以 segment 起点构造局部窗口。
3. 根据 segment 终点在局部窗口中的主方向生成标签。

标签空间：

- `up`
- `down`
- `left`
- `right`
- `stop`

## 为什么这属于弱标注

- 标签不是人工标注。
- 标签直接来自真实走线结果。
- 标签与真实 PCB 走线存在稳定对应关系。

## 为什么不再使用 mock

- 随机标签无法验证模态价值。
- 随机标签会把训练结果变成噪声拟合。
- 第一阶段需要回答的是“任务是否可学”，而不是“骨架是否能吞下随机数据”。

## 质量约束

- 标签必须可复现。
- 样本生成不能依赖随机方向采样。
- `train`、`val`、`test` 必须按 `board-level` 划分。
