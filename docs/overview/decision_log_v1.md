# AIEDA v1 决策日志

## 2026-03-07

### 决策 1

第一阶段统一口径为：

`先验证模态与训练可行性，闭环 / patch / schema 后置。`

原因：

- 当前最大风险不是“闭环没有做完”，而是“任务、数据和训练主链还没收敛”。
- 只有先把同任务可比较实验跑通，后续闭环设计才有依据。

### 决策 2

第一阶段只保留一个主任务：

`LocalRouteChoice-Lite`

原因：

- 多任务并行会让问题定位失真。
- 单任务最容易验证模态价值和后端角色。

### 决策 3

标签来源改为确定性弱标注：

- 直接从真实 `track segment` 派生方向标签。
- 不再接受随机 mock 标签作为正式训练数据。

### 决策 4

数据切分采用 `board-level`：

- 同一块板子的样本不能同时出现在 `train`、`val`、`test`。
- 小样本 fixture 也保持这个规则。

### 决策 5

`DeepSeek-VL` 和 `Janus` 当前只作为实验后端接口占位：

- 不进入正式训练主链。
- 不伪装成可训练主干。
- 当前角色是同任务推理占位和后续实验接入点。

### 决策 6

运行面以 Windows Python CLI 为主：

- `README` 只保留 Python 命令主流程。
- 不再依赖 `.sh` 作为默认入口。
